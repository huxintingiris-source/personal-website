美团 AI 数字员工项目梳理：

这个员工的名称叫小慧，有真实人物形象

1. 项目的目标
   想要从 0 到 1 打造一个 AI 数字员工，去升级我们开店宝后台已有的“店小宝”这个角色。

2. 对商家的价值
   可以降低商家的经营门槛，解决他们实际的线上经营痛点，用 AI 去赋能业务。

我负责的工作内容主要有三个板块：第一个是调研，第二个是AI功能建设，第三个是新场景的接入。完整覆盖了这个 AI Agent 搭建的全链路，从需求调研到功能实现再到场景推广。

其中，在前期调研板块又分为两块：竞品调研和商家调研。

在竞品调研这一块，主要目的是为我们的产品提供可借鉴的思路，找到对项目有启发的点，并确认需求的真伪。我主要从以下三个维度入手：

1. 功能维度：我们核心关注竞品的 AI Agent 提供了什么样的能力。并且更加深入地去拆解它们具体是如何分布处理的，使用了什么样的模型，以及具体的步骤有哪些。当然，我们不仅列举了竞品的一些功能，也深度分析了什么类型的用户（例如新商还是老商）在什么场景下会使用这些功能。

比如：
1. 参考相关资料
2. 做出优化主图等动作
3. 给出一些相似问题

2. 交互维度：我们会看有哪些可以借鉴的产品功能形态，这里又分为两个维度：

1. 入口的维度
   我们会去看在除了对话框之外，还有哪些形式去引导商家进入我们的 AI 数字员工，例如“猜你想问”。

2. 对话和任务完成流程的板块
   我们会去看它是怎么样去引导商家做多轮对话的，比如说可能会给一些选项等。

3. 用户价值维度：我们会评估产品功能对用户是否真的有用，以及是否能提高用户的留存和付费意愿。


介绍完竞品调研的思路，我再详细展开介绍一下我们的竞品。我们主要调研参考的是三个产品：抖音来客、淘宝千牛和 1688 的经营助手。

接下来我们进入商家调研部分。调研的主要目的，是想明确产品的用户画像，让我们产品定位更精准，而不是大而全，满足所有商家的所有需求。

核心内容我分为三个维度：
1. 用户维度：了解哪些商家会为该功能买单，以及他们的接受度。
2. 场景维度：分析商家的特点、痛点，以及在什么场景下会产生需求。
3. 反馈维度：了解商家对当前功能的评价，以及他们想要增加哪些功能。

在执行层面，我们融合了电话访谈、问卷调研、视频会议及线下访谈等多种形式。调研得出的关键结论主要有以下两点：

1. 灰度上线行业的选择
我们将首先选择按摩足疗行业的商家进行功能灰度上线，原因有三：
   (a) 该行业商家高度依赖线上渠道进行经营。
   (b) 行业体量大，且问卷结果显示他们对 AI 的接受程度较高。
   (c) 该行业商家普遍需要指导，代运营模式在群体中比较普及。

2. 商家真实痛点的挖掘
我们最初总结的痛点是“经营难、成本高、流动性大”，但后来发现真正的痛点主要集中在以下三个方面：
   (a) 新商冷启动：商家对规则或活动营销不了解，过去主要靠销售在线下牵引。我们希望通过 AI Agent 替代这部分人力成本。
   (b) 获客难。
   (c) 营销推广。

针对获客和推广这两个相对“无解”的问题，我们调整了产品方向，将定位修改为：针对新入驻的中小商家（尤其是一到三线城市的中青年老板及店员），推出一款帮助他们在美团上智慧经营的助手。

目前总结出的核心产品功能主要分为三块：
1. 经营诊断：属于基础功能。
2. 营销介入：针对获客难、推广难及不了解活动营销等痛点，提供“生图”以及“特团和神会员”场景的介入。
3. 上单组品：在组品阶段，由 AI 辅助完成改团单和上单等功能。

那么在场景上，我们主要瞄准了三个场景：

1. 新商的冷启
   我们会给 Agent 接入一些 RAG 经营规则，包括如何选择商品、如何进行店铺装修等等。

2. 特殊节日的大促阶段
   商家会有更新门店装修的需求，我们会利用生图工具帮他们做一些改进。

3. 评价的分析与回复阶段
   这个阶段我们做得相对比较好，AI 完全可以分析客户评价并做出一个相对完整的回复。


在 AI 功能建设这一块，整体流程如下：

1. 选模型
2. Prompt 调优
3. RAG 知识库接入
4. 制定评测标准并对模型进行评测
5. Query 走查，通过查看一些 Bad Case，引导我们的产品进行迭代

在 query 的走查过程中，我发现 prompt 还有可以优化的空间，于是主动承接了这部分工作，并推进了改版后的 prompt 上线。

最终效果非常显著：
1. 模型的幻觉率大大降低，降到了 20% 以下。
2. 在与研发或算法等部门合作时，建立了一个清晰的标准。

在 prompt 调优的过程中，我当时发现主要难点在于效果难以评估，因此我制定了一套评测标准和大批量调优的 SOP。
我在做 Prompt 调优的过程中，发现一个卡点是调优后的结果难以量化，且缺乏验证过程。
于是我主动与研发沟通交流，搭建了一个工作流，大批量去跑调整后几版的 Prompt 生成效果并进行比对，同时利用制定的评测标准来评价生成的结果。
这套 SOP 的核心逻辑是：
在设定的 Batch 范围内（比如 100 条数据），调优后的 100 个生成结果必须全部满足评测标准。只有在评测标准认为“好”的基础上，我们才会通过该次调优。相比以前可能存在的“随机调优”，这种方式更加严谨和标准化。

具体我是怎么做这个 Prompt 调优的呢？首先一定要明确我们调优的目标。当时我们发现原版本主要有两个不足：

1. 模型会有一些幻觉和 Bad Case。
2. 根据调研结论，我们需要一些更落地的建议。

这就是我调优的核心方向。我首先明确了优化后想要达到的预期效果，具体包含三个部分：

1. 店铺的核心数据：比如金额、曝光、访问、下单量、推广竞争圈等等。
2. 诊断核心问题：这是非常关键的，我们要解决的就是避免让模型给出一些无关紧要的问题。
3. 给出可落地的建议：针对核心问题提供实际建议，后续我们可能还会考虑增加更多功能，比如支持一键完成。

在目标明确之后，我针对 Prompt 优化的具体执行步骤如下：

1. 学习通用的知识。
2. 收集并制作一个“黄金测评集”。
3. 明确评测标准：制定一些可量化的维度，目的是看到实质性的优化进展，比如将 Bad Case 率下降到 5% 以内。

关于维度的部分，我们当时写了非常多版，也在不停地对齐。我主要讲一下思路。

在一级维度层次方面，主要有三个维度：通用性、准确性和有效性。

1. 通用性
   在通用性方面有很多二级维度，比如安全合规、不暴露竞争对手的数据等。

2. 准确性
   这里又有几个细分维度：
   (a) 数据的准确
   (b) 问题的准确
   (c) 建议的准确

3. 有效性
   我们可能会关注：
   (a) 问题与建议是否相关
   (b) 建议是否可落地
   (c) 是否提供了一些具有惊喜性的建议（比如建议商家尝试新的商品组合）


在这个项目当中我比较满意的一个功能：经营诊断。

之所以对这个功能比较满意，是因为它核心解决了商家面临的两个痛点：
1. 读不懂数据
2. 看了数据之后不知道该如何优化，缺乏行动建议

这也是我们在前期调研和业务认知中发现的问题。相对来说，AI 能很好地适配这个场景并发挥作用。这也是我在实习过程中参与推进的一个产品功能。

总结一下，这个功能有明确的用户痛点和使用场景，并且技术也比较成熟。

在这个过程当中，其实最大的困难是评测标准的制定
