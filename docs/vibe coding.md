Vibe coding经验
【Bilibili字幕跳转谷歌插件】
第一次尝试——以失败告终
* 做法：直接把我的需求告诉ai让它生成一段prompt拿去跟coding agent开发
* 总结原因：问题出现在明确需求上，不能直接让agent开发，做错了之后再改怎么都不对，context太乱了
* 解法：
    1. 先在chatbot里把需求对清楚（小tric：让ai反问，把填空题编程选择题，请问我10个问题确认你已经完全理解了我的需求）拿到很完善的prompt之后再让agent开发，顺利多了
    2. 大需求拆解成小需求

第二次尝试——50%成功但修bug失败
学会分步拆解，明确每一步的交付标准
第1步：准备可运行的“空插件”
目标：能在 bilibili.com 注入一个小面板，证明扩展环境跑通（不碰字幕、不碰跳转）。
交付标准：打开B站普通视频页，页面右侧出现“视频字幕（收起态）”壳子。
第2步：实现“点击时间戳跳转”
目标：先不解析字幕，只做一个最小列表（写死几条时间戳），点击后能跳转并播放，验证核心交互链路。
明确交付标准：
* 右侧显示3条假数据（例如 00:00:10 / 00:01:00 / 00:02:30）。
* 点击任意一条：视频跳到对应时间，并尝试开始播放。
第3步：把写死时间戳改为字幕+点击跳转
拉取 subtitle_url JSON 并渲染“时间戳+单句”
继续尝试——MVP版本功能成功！

# 总结SOP
第一步（强烈建议先做）：通过与AI对话头脑风暴理清需求，明确澄清各种模糊说辞
第二步：生成完整PRD尽量采用编程的专业术语
第三步：大需求拆分，让AI一步步来开发
第四步：先生成第一部分的提示词并进入开发，确保这一部分完成后再进入下一个模块的开发
循环如此步骤，这样尽管某次开发不成功，bug无法修复，也可以回退到上一步重新开发
MVP上线之后不断迭代更新优化

AI DEBUG技巧：让AI排查问题问它需要自己提供哪些信息？比如bug网页的开发者工具页面，执行电脑图形界面的操作或者访问某些路径等

# 写PRD prompt
我现在想要开发一款chrome插件，帮我在看bilibili视频的时候，页面右侧清晰的展示所有的文案，并且实现点击某一句话就可以跳转到对应的原视频位置。为了撰写逻辑清晰的PRD需求文档，你先给我提10个问题澄清一下需求。

请直接输出：PRD（目标/范围、用户故事、功能清单、交互说明、异常与提示文案、验收用例、数据结构、权限与合规说明）

现在你是我对接的开发，我要跟你进行需求评审，请你向我提问，保证已经完全理解我的需求
